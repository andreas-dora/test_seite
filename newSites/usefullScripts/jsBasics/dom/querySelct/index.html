Die Methode spukt einen Unübersichtlichen String ausKlassen aus auf 'btn' zutrift'

DOMTokenList(2)
            0: "btn"

QuerySelectorAll - Eventbubbling - und der HTML Template-Tag

Unsere kleine Gästelisten-App (grauer Conatiner) ermöglicht es auf Knopf (add) neue Positionen hinzuzufügen. Aus einem template-tag holt sich die App die den HTML-Code und kopiert ihn in den App-Container. Da die neuen Elemente (Pro Eintrag ein Input-Flrd und ein Deletb-Button) noch nicht Existent waren könnnen wir die mit gängigen Abfragen nicht erreichen.

Ein einiger Eventlistner auf dem ganzen App-Container lösst das Problem. Jeder Klick inden Kontainer löst nun eine JS-Funktion aus, die an erster Stelle zu Fragen hat was da angeklickt wurde.

Ein nicht zu unterschätzender Vorteil dieser dieser Methode ist, dass die gesamte GL-App mir mit nur einem EventListener auskommt. Egal wie lang die Gästeliste wird unser Skript bleibt unverändert knackick und unsere Seite so schön geschmeidig.


Bubbling und Capture Phase

Jedes DOM-Event, ganz egal ob click-, hover- oder Thunde-Dom-Event, in der 'Bubbeling'-Phase die von seinem Ursprung (e.target) ausgehend die gesamte Verzeichnisstruktur hinauf, bishin zu body, html und window. Was bedeutet das wir unseren Event-Listener auch gut an der nächsten Dom-Etage aufstellen können. Im Bubbeling wird das Klingeln also immer von Child zu Parent getrangen. Das gegenstück zur Blubber-Phase ist die eigentlich vorangehende Capturing-Phasse, sie wird die VErszeichnis-Struktur hinunter von PArent zu Child geleitet.

      <button class="btn del">Delete</button>
    </div>

Unsere vom App-Kontainer ausgelösste Lösch-Funktion empfängt das Event 'e'. Mit e.target wird der tatsichliche Auslöser der Events abgefragt. Hat dieser Klasse 'del' (delete) wird es ernst. Ein Blick zurück auf das HTML zeigt, dass die zu lÖchenden Elemente (Label, Input und BUtton, gemnsam in einem KOntainerchen sitzen. Das macht die Sache leicht: Unsere Löschfunktion muss nur das .parentElment schnappen und kann dann die drei Teilchen gemeinsam "re-movern".


  document.querySelector('#app').addEventListener('click', deleftFun)
  function deleftFun(e){
    if(e.target.classList.contains('del')){
      e.target.parentElement.remove();
    }
  }
  document.querySelector('#addBtn').addEventListener('click', addBox);

Und dies ist der Teil des Skripts mit dem das PAket aus dem Template-String kopiert wird.



  function addBox(){
    var temp = document.getElementById('temp').content;
    var copy = document.importNode(temp,true);
    document.getElementById('app').appendChild(copy);
  }

HTML Elemente hinzufügen

Vier Arbeitsschritte sind notwenig um ein HTML hinzuzufügen

  1. createElement()
  2. createTextNode()
  3. appendChild()
An dieser Stelle soll das neue Elemt eingepflanzt werden:
const box = document.querySelector('.zielBox');

const box = document.querySelector('#zielBox');
var newElement = document.createElement('a');
var newContent = document.createTextNode("THIS IS NEW");
newElement.appendChild(newContent);
box.appendChild(newElement);

Dieser Code ersparrt einen Arbeitsschritt

const box = document.querySelector('#zielBox');
var newElement = document.createElement('a');
newElement.append('A NEW LINK');
box.appendChild(newElement);

Fügt man dem Anchor noch das href hinzu ist auch schon alles getan:

newElement.setAttribute('href','www.castel-of-dolphins.de');

HTML template Tag

Der Inhalt des tempalte-Tags wird im Browser nicht dargestellt. Hier schllummert meine Vorlage:

    
    <template id="temp">
      <div class="inputBox">
        <label for="Gast">Gast</label>
        <input type="text" placeholder="Name">
        <button class="btn del">Delete</button>
      </div>
    </template>

Mit ein vier Zeilen Javascript wird der Inhalt des Template vervielfälltigt.



function addBox(){
 var temp = document.getElementById(‘temp’).content;
 var copy = document.importNode(temp,true);
 document.getElementById(‘app’).appendChild(copy);
}
document.querySelector(‘#addBtn’).addEventListener(‘click’, addBox);
