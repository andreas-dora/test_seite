Datentypen für rationale Zahlen

Rationale Zahlen können durch einen Bruch dargestellt werden. Rationale Zahlen können als Fixkomma- oder Fließkomma Zahlen dargestellt werden. Fixkomma Zahlen werden benutzt wenn es großer Genauigkeit bedarf.

Für Fixkomma Zahlen stehen die Typen DECIMAL(p,s) und **(p,s). In mySQL unterscheiden sich die Beiden Typen nicht.

  • P = Precision (Gesamtlänge)
  • S = Scale (Anzahl der Stellen hinter dem Komma)

Daten werden nach dem ISO 8601 Standard gespeichert: yyyy-mm-dd

Zeitangaben: hh:mm:ss.f f ist Platzhalter für den angebrochene Teil einer Sekunde

TIME wird verwendet um Zeitintervalle bis zu 838 Stunden darzustellen.

Datentypen für Strings

Datentyp Maximale Zeichenkettenlänge
CHAR(L) 255
VARCHAR(L) 65.535 (seit mySQL 5.0.3)

PostgreSQL speichert bei beiden Typen maximal 10.485.760. MS SQL Server kann bei beiden Typen maximal 8000 Zeichen speichern.

CHAR(100) hat den Nachteil, dass nicht genutzte Stellen mit Leerzeichen aufgefüllt werden. VARCHAR(100) hat ebenfalls eine max Länge von 100 Byte allerdings werden nicht benutze Stellen nicht mit Leerstellen aufgefüllt.
