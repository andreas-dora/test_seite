SQL Datentypen konvertieren

SQL bietet eine CAST Konvertierungsfunktion. CAST kann bei allen UPDATE-Operationen zum Einsatz kommen.

Typenkonvertierung kann notwendig sein bei:

  • Mathematischen Operationen
  • Beim Einfügen INSERT und UPDATE
  • Beim Vergleichen einer Bedingung in einer WHERE-Klausel
  • Bei ORDER BY
  • Bei Anwendung einer UNION-Klausel
  • Beim Aufrufen von Skalierfunktionen

Wird eine Konvertierung vorgenommen um zu Rechnen oder Inhalte zusammen zu fügen muss man sich immer an demDatentyp mit dem höheren Wertebereich orientieren. Gibt man einer Datenbank keine weiteren Anweisungen zur Konvertierung spricht man von einer impliziten Konvertierung. Wird die CAST Funktion (in der man der Datenbank seine Wünsche Mitteilt** verwendet, ist das eine explizite Konvertierung

SELECT CAST(5 AS DECIMAL(6,3)) AS kovertierungsergebnis;

Ergebnis: 5.000 (Eine 5 mit drei Kommastellen).

Aufrunden mit Cast funktioniert nicht bei mySQL

SELECT CAST(3.4 AS INTEGER) AS ganzezahl;
